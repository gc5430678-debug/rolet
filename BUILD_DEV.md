# بناء تطبيق Rolet للتطوير (Dev Build) — إشعارات رسائل حقيقية

لتجربة إشعارات الرسائل الحقيقية (اسم المرسل + صورته) حتى عند إغلاق الهاتف، تحتاج **Development Build** وليس Expo Go.

## المتطلبات

- حساب [Expo](https://expo.dev) (مجاني)
- [EAS CLI](https://docs.expo.dev/build/setup/) مثبت
- جهاز أندرويد فعلي (أو محاكي للاختبار الأساسي)

## الخطوات

### 1. تثبيت EAS CLI

```bash
npm install -g eas-cli
```

### 2. تسجيل الدخول

```bash
eas login
```

### 3. بناء نسخة التطوير (Android)

```bash
cd rolet
eas build --profile development --platform android
```

سيُطلب منك إعداد المشروع إذا كان أول مرة. اختر الافتراضيات.

### 4. تنزيل وتثبيت الـ APK

بعد انتهاء البناء (حوالي 10–20 دقيقة):

1. ادخل إلى [expo.dev](https://expo.dev) → مشروعك → Builds
2. حمّل ملف الـ APK
3. ثبّته على جهازك (قد تحتاج تفعيل "مصادر غير معروفة")

### 5. تشغيل التطبيق

```bash
npx expo start --dev-client
```

امسح رمز QR من التطبيق المثبت (وليس Expo Go).

## كيف تعمل الإشعارات

- عند تسجيل الدخول، يُسجّل التوكن في الخادم
- عند إرسال أي مستخدم رسالة لك، يُرسل إشعار يحتوي على:
  - **العنوان:** اسم المرسل
  - **النص:** محتوى الرسالة (أو "رسالة صوتية" / "صورة" / "هدية")
  - **الصورة:** صورة المرسل (إن وُجدت)
- الإشعار يصل حتى لو كان الهاتف مغلقاً أو التطبيق في الخلفية

## ملاحظات

- **Expo Go** لا يدعم إشعارات الدفع الحقيقية في SDK 53+ — يجب استخدام Dev Build
- تأكد أن الخادم (الباك اند) يعمل ومتصل بالإنترنت
- رابط الصورة في الإشعار يجب أن يكون عاماً (مثل `https://...`) حتى يتم تحميلها
